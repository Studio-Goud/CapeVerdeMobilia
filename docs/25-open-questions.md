## 3. Validation plan / next steps

### 3.1 Sequencing (RECOMMENDATION)

1. **Legal/fiscal confirmation first (highest consequence).** Engage local counsel + tax advisor to confirm assumptions #8, #9, #10, #11, #21 and the source items flagged "legal/tax verification required." This underpins all buyer-facing content.
2. **Behavioural validation** (`24-validation-interviews.md`). Run interview waves across priority segments to test assumptions #1, #2, #4, #5, #6, #7, #20, #25, #26; score and rank.
3. **Government soundings.** Approach one municipality for a Model 1 pilot to test assumptions #16, #17, #18. **Government confirmation required.**
4. **Brand/legal clearance.** Run domain/trademark/handle checks (#23) via official registrars and IP counsel — never announce the name before clearance (see `21`).
5. **Market sizing.** Confirm #13, #14 with official census/INE and market data; build bottom-up SOM/SAM/TAM.
6. **Ops/unit-economics instrumentation.** During pilot, measure cost-per-onboarding, trust-loop and retention to test #19, #20.
7. **Finance/FX.** Confirm the peg (#22) and lock pricing in € terms.

### 3.2 Cadence & governance

- Review this register at the end of each validation wave; update **Status** (🔴/🟡/🟢) and add findings + dates.
- Every marketing/pitch claim must trace to a row here; if a claim has no validated row, label it **ASSUMPTION/HYPOTHESIS** in the deliverable.
- Keep the "accessed 2026-07-20" sources under periodic re-check, since digital-gov and tax items change quickly.

### 3.3 Definition of "validated"

An assumption moves to 🟢 only when: (a) confirmed by an authoritative source **or** professional (legal/tax/official), or (b) supported by a sufficient, non-contradictory behavioural sample — with the evidence and date recorded in this file.
